# Test Report – Huntd Application

---

## 📌 Summary

Testing was performed on the Huntd application across web and mobile platforms.

The focus was on validating core user flows, role-based access, and application behavior.

---

## 🧪 Scope

- web application (desktop & mobile browser)
- mobile application (Android)

---

## 🔍 Tested Areas

- authentication (sign in / sign up)
- profile management
- job search and filters
- role switching (Candidate / Recruiter)
- chat functionality
- navigation and UI elements

---

## 🐞 Defects Summary

During testing, multiple defects were identified:

- functional issues (filters, chat features)
- permission issues (access restrictions)
- UI/content issues (missing elements in footer)
- data logic issues (incorrect sorting)
- consistency issues (naming inconsistencies)

---

## 📊 Results

- test cases executed: (approximate)
- defects found: multiple (low, medium, high)

---

## ⚠️ Risks

- incomplete or evolving requirements
- differences between web and mobile implementations
- limited access to tools (TestRail trial expired)

---

## ✅ Conclusion

The application requires fixes before production release, especially in:

- filtering functionality
- chat features
- consistency of UI elements
