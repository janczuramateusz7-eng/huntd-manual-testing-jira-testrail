# Task Distribution – Huntd Project

---

## 📌 Overview

Testing tasks were distributed among team members to ensure full coverage of application features across web and mobile platforms.

Each tester was responsible for specific functional areas.

---

## 👥 Team Responsibilities

### Web Application
Testers were assigned to cover different areas:
- authentication
- profile
- jobs
- filters
- navigation
- chat functionality

### Mobile Application
Testing tasks were distributed for:
- authentication
- profile
- job browsing
- notifications
- navigation

---

## 🧪 My Responsibilities

As a QA tester, I was responsible for:

- creating test cases in TestRail
- testing assigned features
- reporting bugs in Jira
- validating fixes (retesting)
- performing exploratory testing
- contributing to decomposition and test design

---

## 🔁 Workflow

The team followed a structured workflow:
1. analyze requirements
2. create decomposition
3. assign features to testers
4. create test cases
5. execute tests
6. report bugs in Jira
7. retest fixes
8. prepare test report

---

## 🎯 Goal

The goal of task distribution was to:
- ensure full test coverage
- avoid duplicated work
- improve testing efficiency
- enable parallel testing across features
