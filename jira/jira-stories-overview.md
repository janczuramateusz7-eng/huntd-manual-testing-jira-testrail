# Jira Stories Overview

---

## 📌 Overview

Jira was used to manage the testing process and track work items during the project.

---

## 📋 Story Structure

Each feature was represented as a separate user story.

Examples of features:
- authentication
- profile management
- job search
- chat functionality
- filters

---

## 🧩 Workflow

The workflow included:
1. creating stories based on requirements
2. assigning tasks to testers
3. executing tests
4. reporting bugs
5. tracking progress on the board

---

## 🐞 Bug Tracking

All defects were reported in Jira with:
- summary
- steps to reproduce
- expected vs actual result
- severity and priority
- screenshots or recordings

---

## 🎯 Goal

The goal of using Jira was to:
- organize testing work
- ensure feature coverage
- track defects efficiently
